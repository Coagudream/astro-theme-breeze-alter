# 渲染管线

## Input

输⼊汇编器是负责接收和组织数据的那⼀部分场景中存在的对象（⽹格）,此过程包括以下信息：
- 顶点
- 索引
- 法线
- 切线
- 坐标
- 其他⼏何属性
所有这些信息都作为数值存储，并以结构化的⽅式组织，以便 渲染管线能够正确地解释它。这些结构化数据随后被处理，最终⽣成屏幕上显⽰的 3D 对象。
## rendering mode

  渲染模式的选择决定了将使⽤哪种渲染技术 项⽬（Forward+、Forward Mobile 或兼容模式）。这⼀选择直接影响 性能和视觉质量，因为它定义了图形的处理⽅式以及将使⽤的渲染 API（Vulkan、Direct3D、OpenGL ES 等）。根据 所选模式， 某些图形效果和优化要么可⽤，要么受限，从⽽影响着⾊器如何与场景交互
## vertex processing stage

顶点处理阶段，由 vertex()⽅法表⽰，是基础之⼀ 着⾊器中的阶段。在此阶段，从输⼊组装器检索到的数据是 在不同的坐标系中转换，遵循以下顺序：
-  对象空间：模型的局部空间，顶点相对于⾃⼰的轴⼼定义。
-  世界空间：根据对象在场景中的位置变换顶点。
-  视图空间：根据相机的视⻆调整顶点，决定它们如何出现在观察者眼中。
-  投影空间（裁剪空间）：将顶点投影到⻬次坐标系中，为它们做准备 为下⼀阶段。
-  标准化设备坐标：执⾏除以 W 操作，将坐标标准化到 在各个轴上介于-1.0 和 1.0 之间。
-  屏幕空间：将顶点分配到物体在屏幕上的像素坐标处光栅化并渲染。
这个过程确保 3D 场景正确地投影到 2D 表⾯（屏幕）上， 将多边形模型转换为屏幕上的可⻅图像

## rasterization

光栅化阶段，⼏何基元——例如三⻆形、线条或点，被转换为屏幕上的⽚段（潜在的像素）。这个过程决定了物体哪些部分将是可⻅的以及它们将如何在最终图像中分布。

在此阶段应⽤了深度测试——这是⼀个决定性的关键机制 基于⽚段相对于相机的距离来决定其可⻅性。这个测试可以 将每个⽚段的深度与 Z 缓冲区（或深度缓冲区）中存储的值进⾏⽐较， 丢弃被其他物体遮挡的部分。这确保了只有最近的可⻅部分渲染表⾯时，可防⽌重叠伪影并提升场景真实感

## fragment stage

⽚段处理阶段，在那⾥它的最终颜⾊在显⽰为屏幕上的像素之前进⾏计算。在这个阶段，关键视觉效果 属性都是确定的，包括光照、纹理、透明度和反射。此外，每个⽚段在 GPU 中独⽴处理，从⽽能够⾼效执⾏复杂在保持⾼性能的同时进⾏计算。


## Blending and Post-Processing

应⽤混合和后期处理阶段，包括视觉效果 在将图像渲染到屏幕上之前。在这个阶段，透明度、颜⾊ 混合、泛光、抗锯⻮和⾊彩校正等元素被处理。这些效果增强了 视觉效果并优化场景的最终外观，确保更加精致和沉浸

# 顶点函数

顶点函数vertex()会作用在每一个顶点之上-----GPU并行
所以会有统一的VERTEX内置变量用来表示全部的顶点坐标
``` gdshader
render_mode skip_vertex_transform
//跳过顶点变换阶段，使自己手动控制顶点变换阶段（在渲染管线之中）
```

在着⾊器中，矩阵主要⽤于将顶点位置从⼀个空间转换到 另⼀个空间。
- 模型矩阵（MODEL_MATRIX）：将数值数据转换为世界空间
- 视图矩阵（VIEW_MATRIX）：将世界空间转换到视图空间
- 投影矩阵（PROJECTION_MATRIX）: 将视图空间转换到投影空间（裁剪空间）
- INV_VIEW_MATRIX : 视图空间到世界空间的转换

> 在Godot 中，你会找到⼀个名为 MODELVIEW_MATRIX 的矩阵。这个矩阵结合了模型 (MODEL_MATRIX) 和视图 (VIEW_MATRIX) 变换。因此，它变换⼀个点直接从局部空间转换到视图空间。

# 片段函数
片段函数fragment()会对图片的每一个像素使用（相当于使用color循环遍历每一个像素）-----GPU并行
``` gdshader
vec3 view_to_tangent(vec3 tangent,vec3 bitangent, vec3 normal, vec3 view)
{
	float t = dot (tangent,view);
	float b = dot (bitangent,view);
	float n = dot (normal,view);
	return normalize(vec3(t,b,n));
}
	//dot()函数计算两个向量的点积。在这种情况下，它作⽤于三个维度向量，结果是⼀个标量值，然后将其赋值给变量.(投影)
	//切线，tangent
	//副切线，bitangent
	//法线，normal
	//t、b 和 v 基于相应的基向量——切线、副切线和法线。
	//VIEW 是⼀个从正在处理的⽚段指向摄像机，并且它是在摄像机的参考空间中表达的。这个向量是⾃动的在光栅化过程中从顶点进⾏插值，它特别适⽤依赖于视⻆的效果
```
